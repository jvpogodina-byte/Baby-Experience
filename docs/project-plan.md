# Проект рекомендаций для мамы и новорожденного

## Summary
- Full-stack веб-приложение с публичной частью и админкой.
- Пользователь может смотреть рекомендации по категориям, открывать карточки вещей, читать пояснения, видеть примеры конкретных вещей, оставлять комментарии и сохранять списки.
- Для MVP выбран стек: Next.js + PostgreSQL + Prisma + Auth.js/NextAuth.
- Авторизация: email/password и Google OAuth.

## Key Changes
- Публичная часть:
  - Главная с подборками и навигацией по категориям.
  - Категории: для мамы, для малыша, для дома, для путешествий, для прогулок и расширяемый список.
  - Страница категории с фильтрами и карточками вещей.
  - Страница вещи с:
    - названием,
    - категорией/тегами,
    - объяснением, почему вещь рекомендована,
    - примером конкретной вещи в виде ссылки и/или фотографии,
    - комментариями пользователей.
- Авторизация:
  - Регистрация и вход по email/password.
  - Вход через Google.
  - Комментарии и сохранение списков доступны только авторизованным пользователям.
- Пользовательский кабинет:
  - Сохранённые списки вещей.
  - Избранное и история по категориям.
  - Базовый профиль пользователя.
- Админка:
  - CRUD для категорий и вещей.
  - Управление пояснениями и примерами.
  - Загрузка изображений и хранение ссылок.
  - Публикация через `draft -> published`.
  - Модерация комментариев при необходимости.
- Модель данных:
  - `User`
  - `Category`
  - `Item`
  - `ItemExample`
  - `Comment`
  - `SavedList` / `SavedItem`
  - `MediaAsset`
  - роли админа и статусы публикации.
- UX/контент:
  - Акцент на доверии и практичности.
  - Одна вещь может относиться к нескольким сценариям через теги.

## Test Plan
- Авторизация:
  - регистрация,
  - вход по паролю,
  - вход через Google,
  - ограничения для гостей.
- Контентные сценарии:
  - создание категории,
  - создание вещи,
  - добавление пояснения,
  - добавление ссылки/фото,
  - публикация из черновика.
- Пользовательские сценарии:
  - просмотр каталога,
  - фильтрация по категориям,
  - комментарии,
  - сохранение вещей в список.
- Админские сценарии:
  - редактирование и удаление,
  - скрытие/модерация комментариев,
  - работа со статусами.
- Нефункциональные проверки:
  - мобильная версия,
  - доступность,
  - базовая SEO-структура.

## Assumptions
- Берём Next.js full-stack как самый быстрый путь для MVP.
- Google login = Google OAuth.
- Фото и ссылка на пример вещи хранятся в одной карточке, с возможностью расширить это позже.
- По умолчанию поток публикации: `draft -> published`.
- Отдельную модерацию комментариев можно добавить позже, если потребуется.
